# GNSSR Front-End Host

用于承载 GNSS-R 前端上位机设计资料与后续实现代码的仓库。

当前仓库的第一阶段内容以设计规格为主，核心文件为 `gnssr_windows_host_agent_spec.yaml`。根据现有规格，项目目标是构建一个运行在 Windows 10/11 x64 平台上的 GNSS-R 上位机程序，面向 FX3 USB 数据链路与前端串口控制场景，推荐技术栈为 `C# + .NET 8 + WPF + MVVM`。

## 当前内容

- `gnssr_windows_host_agent_spec.yaml`：Windows 上位机设计规格说明

## 仓库用途

- 沉淀上位机需求、接口约束与架构设计
- 作为后续桌面端工程代码的统一版本库
- 为联调、测试、采集流程和发布打基础

## 计划范围

首个 MVP 以“稳定连接、可视状态、可控采集、可靠写盘”为目标，重点覆盖以下能力：

- FX3 USB 设备枚举与连接
- EP0 控制命令调用
- Bulk IN 数据接收
- 串口控制与状态读取
- 原始数据落盘与基础日志
- 简洁可用的桌面端状态界面

## 推荐开发环境

- Windows 10 / Windows 11
- .NET 8 SDK
- Visual Studio 2022 或 JetBrains Rider
- Git
